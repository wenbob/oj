# 学生使用说明

这份说明给学生使用。你只需要会登录、找题、写 C++ 代码、提交、查看结果和参加模拟考试。

## 1. 如何登录

1. 打开平台登录页：

```text
/login
```

2. 输入老师提供的用户名和密码。
3. 点击登录。
4. 登录成功后会进入学生首页。

如果提示用户名或密码错误，请先确认是否输错；仍然无法登录时，联系老师重置密码。

## 2. 如何进入日常刷题

登录后，在学生端顶部菜单点击：

```text
日常刷题
```

也可以直接访问：

```text
/student/problems
```

这里会显示日常练习题目列表。

## 3. 如何按分类找题

题目列表顶部会显示分类，例如：

```text
全部
基础语法
条件判断
循环结构
数组
字符串
函数
综合练习
```

使用方式：

1. 点击“全部”可以查看所有题目。
2. 点击某个分类，只显示该分类下的题目。
3. 每道题会显示题目标题、难度和分类。
4. 如果题目很多，可以使用上一页、下一页翻页。

## 4. 如何查看题目

在题目列表中点击题目的进入按钮，会打开题目详情页。

题目详情页会显示：

- 题目标题
- 难度
- 分类
- 题目描述
- 输入格式
- 输出格式
- 样例输入
- 样例输出
- 数据范围
- C++ 代码编辑器

样例标题在代码框外面，代码框里面只是真正的输入或输出内容。复制样例时，不要把“样例输入 1”这样的标题复制进去。

## 5. 如何使用 Monaco Editor 写 C++ 代码

题目详情页中的代码编辑器类似 VS Code。

支持：

- C++ 语法高亮
- 行号
- Tab 缩进
- 自动缩进
- 括号和引号自动补全
- 括号匹配
- 粘贴代码时保留格式
- 暗色主题

第一次打开题目时，如果没有草稿，会显示默认 C++ 模板。

如果你修改过代码，平台会自动把草稿保存到浏览器本地。刷新页面后，之前写的代码通常还会保留。

## 6. 如何提交代码

1. 在编辑器中写好 C++17 代码。
2. 点击“提交代码”。
3. 等待评测完成。
4. 页面会显示评测状态、通过测试点数量、运行时间和本次程序输出。
5. 如果需要看每个测试点的输入、你的输出和标准输出，点击“查看详情”。

注意：

- 提交前请确认代码能从标准输入读取数据。
- 不要在代码里写死样例答案。
- 不要使用需要网络、文件系统或系统权限的代码。

## 7. 评测状态是什么意思

### Accepted

答案正确，所有测试点都通过。

### Wrong Answer

程序能运行，但输出和标准答案不一致。

提交结果区域会优先显示第一个错误测试点的“程序输出”，可以先对照题目样例或提交详情里的标准输出排查。

常见原因：

- 算法错了。
- 输出格式不对。
- 多输出了提示语。
- 少了空格或换行。
- 只考虑了样例，没考虑其他测试点。

### Compile Error

代码编译失败。

常见原因：

- 少了分号。
- 括号不匹配。
- 变量名写错。
- 使用了不存在的函数。
- C++ 语法错误。

### Runtime Error

程序运行时出错。

常见原因：

- 数组越界。
- 除以 0。
- 递归太深。
- 程序异常退出。

### Time Limit Exceeded

程序运行超时。

常见原因：

- 死循环。
- 算法太慢。
- 循环次数过多。

## 8. 如何查看日常提交记录

点击顶部菜单：

```text
日常提交
```

也可以访问：

```text
/student/submissions
```

这里会显示你自己的日常刷题提交记录，不会显示其他同学的提交，也不会显示模拟考试提交。

每条记录会显示：

- 题目
- 状态
- 通过测试点数量
- 运行时间
- 提交时间
- 查看详情
- 复制代码
- 继续修改

## 9. 如何复制历史代码

在日常提交记录或提交详情页中，点击：

```text
复制代码
```

复制成功后可以把代码粘贴到编辑器或本地文件中。

如果复制失败，可以进入提交详情页，手动选中代码内容复制。

## 10. 如何继续修改历史代码

在提交记录或提交详情页点击：

```text
继续修改
```

平台会回到对应题目的详情页，并把这次历史提交的代码加载到编辑器中。

加载后可以继续修改，再次提交。

## 11. 如何参加模拟考试

点击顶部菜单：

```text
模拟考试
```

也可以访问：

```text
/student/exams
```

这里会显示老师已经发布的考试。

如果看不到某场考试，可能是：

- 老师还没有发布。
- 考试已经结束。
- 页面需要刷新。

## 12. 如何开始考试

1. 在模拟考试列表中点击进入考试。
2. 查看考试名称、说明、题目数量和考试时长。
3. 点击“开始考试”。
4. 进入考试答题页。

同一场考试只会记录一次开始时间。退出后再次进入，会继续这场考试。

如果已经交卷或超时，就不能继续答题，只能查看结果。

## 13. 如何在考试中提交代码

考试答题页通常分为左右两部分：

- 左侧：考试题目列表和每题状态。
- 右侧：当前题目内容、代码编辑器、提交按钮。

使用方式：

1. 点击左侧题目。
2. 阅读右侧题面。
3. 在编辑器中写代码。
4. 点击提交。
5. 查看当前题目的评测状态。

考试中的提交会记录到考试提交记录，不会混入日常提交记录。

## 14. 如何交卷

考试答题页会显示剩余时间。

完成后点击：

```text
交卷
```

交卷后：

- 不能继续提交。
- 会进入考试结果页。
- 可以查看总分和每题结果。

如果时间到了，系统会自动结束考试并跳转结果页。

## 15. 如何查看考试结果

交卷或超时后，可以进入：

```text
/student/exams/[考试ID]/result
```

结果页会显示：

- 考试名称
- 考试状态
- 开始时间
- 交卷时间
- 总分
- 每题得分
- 每题最高状态
- 每题提交次数

当前计分规则：

```text
每道题只要在本场考试中有 Accepted，就获得该题满分。
否则该题 0 分。
```

## 16. 如何查看考试提交记录

点击顶部菜单：

```text
考试提交
```

也可以访问：

```text
/student/exam-submissions
```

这里只显示你自己的模拟考试提交记录。

## 17. 常见问题

### 登录失败怎么办

先检查用户名和密码是否正确。如果仍然失败，联系老师重置密码。

### 提交后一直等待怎么办

可能是评测队列正在运行，稍等一会儿再看。如果长时间没有结果，请联系老师检查 Docker Judge。

### 为什么样例过了，但提交是 Wrong Answer

样例只是部分测试。系统还有其他测试点。你的程序需要对所有合法输入都正确。

### 为什么不能在考试中继续提交

可能原因：

- 你已经交卷。
- 考试已经超时。
- 老师结束了考试。
- 你还没有点击开始考试。

### 为什么复制代码失败

浏览器可能没有授权剪贴板权限。可以进入提交详情页，手动选中代码复制。

### 为什么我的草稿不见了

草稿保存在当前浏览器本地。如果更换电脑、清理浏览器数据或使用无痕模式，草稿可能无法恢复。

### 可以提交哪些语言

当前平台只支持 C++17。
